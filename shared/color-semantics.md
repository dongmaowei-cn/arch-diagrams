# 颜色语义 · 不要随意混用

模板里 7 种 accent 颜色都有固定语义。**改造时只用对应颜色，不要为了好看交换。**

## 颜色 → 语义

| 颜色 | hex | 语义 |
|---|---|---|
| **clay**  | `#D97757` | 动作 / 数据库 / 强调 / 主品牌色 |
| **olive** | `#788C5D` | 成功 / 主路径 / YES 分支 |
| **rust**  | `#B04A3F` | 失败 / 错误 / NO 分支 / 终态-失败 |
| **plum**  | `#6B5B95` | 外部 / I/O / 消息 / 异步 |
| **gold**  | `#C4994E` | 网关 / 边缘接入 / 注释 |
| **teal**  | `#4A8585` | 横切设施 / 监控 / 链路 |
| **gray**  | `#87867F` | 默认 / 中性 / 顺序流 |

## 选色决策

```
要表达「成功 / 通过 / 主路径」     → olive
要表达「失败 / 错误 / 拒绝」       → rust
要表达「与外部通讯」              → plum
要表达「访问数据库」              → clay
要表达「网关 / CDN / WAF」        → gold
要表达「监控 / k8s / 横切设施」    → teal
其它日常顺序流                   → gray
```

## 节点配色规则

- 节点颜色 = `fill: color-mix(in oklab, var(--{color}), transparent 85~92%)`
- 节点描边 = `stroke: var(--{color})`
- **大原则**：节点颜色匹配它"做什么"，不要匹配它"在哪层"
  - ✓ 支付服务用 olive（主路径）
  - ✗ 第 4 层全部强行 olive

## 暗色模式

模板的 `[data-theme="dark"]` 已经把这 7 个 hex 自动映射到暗色变种 —
**只要你用的是模板里的 hex 或 var()，自动适配。**

切记不要写成：
- ✗ `<rect fill="#788C5D">` 这种硬编码 hex（虽然有 attr selector 兜底，但脆弱）
- ✓ `<rect class="shape">` + CSS 控制颜色（最佳）
- ✓ `<rect style="fill: var(--olive)">` （次佳）

## 反例

- ✗ 用 clay 表达失败（应用 rust）
- ✗ 主路径用 gray（失去强调）
- ✗ 一张图里 7 种颜色全用上（视觉混乱，主路径就不突出了）
- ✗ 给"用户"节点强行配 olive（用户不天然成功；用 gray 即可）

## 一张图的颜色配额建议

```
gray   :主路径以外的节点和边，约占 50-70%
clay   :焦点节点的描边、DB 边、注释引线
olive  :主路径（spine），约占 10-20%
rust   :失败终态 + NO 分支，不超过 3 处
plum   :外部接口/消息节点，不超过 3 处
gold   :网关 + 边缘接入 + WAF
teal   :横切设施层 + 监控边
```
