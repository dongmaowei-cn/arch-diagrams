订单结算时序 · 总览

5 参与者·4 阶段·15 消息·3 fragment

一笔订单从用户提交到银行扣款的完整时序。四个阶段串行：幂等校验 → 发起支付 → 状态轮询 → 异步完成。

关键不变量：同一 idempotency_key 重复请求永远返回同一 order_id；out_trade_no 在 Payment 侧也保证不会重复扣款。

骨架规则：每根激活框宽 10px 居中于 lifeline；消息从一根激活框右/左边缘出发，到另一根激活框对应边缘结束 —— 不画到 lifeline 中心；调用方激活长度 ≥ 被调用方。

How to draw · 骨

时序图的七步骨架

  1. 列出参与者：左到右按发起顺序排 — User → Edge → Internal → External。每个 actor 顶部一矩形 + 下方一根虚线 lifeline。
  2. 切阶段（phase）：用大虚线框把每个业务阶段框起来，左上角标 PHASE n · 名称。阶段是时序图的章节。
  3. 画激活框：每个 actor 在它"正在处理"期间画一根窄竖矩形（宽 10px）覆盖 lifeline。调用方激活长度 ≥ 被调用方 — 调用方必须先激活、后取消激活。
  4. 消息线落在边缘：sync = 实线 + 实心箭头；async = 虚线 + 实心箭头；return = 虚线 + 开口箭头。从一根激活框的边出发，到另一根激活框对应边结束。
  5. self-message 走 U 形：从激活框右边出 → 横走 → 下折 → 横回到激活框右边。
  6. 组合片段 fragment：alt（互斥分支）/ loop（重复）/ opt（条件）— 左上角小斜角标 + 内部虚线 divider 分隔多分支，guard 写在方括号里。
  7. 补语义：折角矩形 = note 注释；lifeline 末端的 ✕ = destroy（连接关闭）；圆圈序号 ① ② … = 强调时间顺序。
