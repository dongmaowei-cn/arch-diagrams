How to draw

状态机的八步骨架

  1. 定义起止：每张图有一个 initial（实心圆），可以有多个 final（同心圆）。异常退出用 terminate ×。
  2. 抽出主干状态：每个状态用 名词 命名（PAID, SHIPPED），不是动词。简单 → simple state；含子状态 → composite + name-tag。
  3. title + 横线 + actions：状态框分两段 — 标题（名词）与横线之下的 entry / do / exit / defer / 内部 transition。
  4. 三段式 transition：trigger [guard] / action。trigger 必填；guard 与 action 可选。completion transition 是无标签的（子状态自动退出）。
  5. 时间与变化事件：用 after(d) 表示超时（不要写 timeout）；用 when(c) 表示条件变化触发。
  6. 组合状态 + history：用虚线大框 + 左上 name-tag 标记 composite；内嵌 H（回当前层）或 H*（递归回最深层）。
  7. 判断分流：junction（实心小圆 · 编译期静态分流）vs choice（空心小菱形 · 运行时动态分流）；二者形状不同。
  8. 非 UML 扩展打 [ex]：业务专属字段（SLA、retry-counter、priority）属于扩展，挂金色 [ex] 小角标标注 — 让阅读者一眼分清"标准 UML"还是"我们自己加的"。
