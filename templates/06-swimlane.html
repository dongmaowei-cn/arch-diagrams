How to draw

泳道图的八步骨架

  1. 定义 Pool：每个参与方（一个公司/系统）画一个 pool。同 pool 内的部门用 lane 细分；不同公司必须是不同 pool。
  2. 抽出 Lane：每个职能/部门 = 一条 lane。常见错：把"用户"放在 pool 边缘（应为独立 pool 才对，但简化主图常合并）。
  3. 找 start / end：每个 pool 至少一个 start（单细圆）+ 一个 end（粗双线圆）。多 end 表多种结果。
  4. 任务用 task 类型：User · Service · Send · Manual · Script 各有图标 — 不要全画 Service 当万能盒子。
  5. 分支用 gateway：XOR ×（if/else，只走一条）· AND +（同时全走）· OR ○+◇（满足条件的都走）。
  6. 连线选对：同 pool 内 = sequence flow（实线）· 跨 pool = message flow（虚线 + 圆头）。不要混用。
  7. 数据用 data object / store：临时单据 = document 形状 · 持久库 = cylinder。用关联虚线连，不是 sequence flow。
  8. 非标记 [ex]：SLA / 超时阈值 / 异步并行标记是业务扩展，挂金色 [ex] 角标 — 与标准 BPMN 区分开。
