怎么画 · How to draw

系统架构图的骨架

  1. 定层数：典型 5-7 层。前端 / 接入 / BFF / 业务 / 中间件 / 数据，必要时把横切层（K8s/监控/网格）单独成层。
  2. 定方向：用户请求自顶向下，顶 = 用户最近、底 = 数据最近。横向才是同层服务之间的协作。
  3. 每层一个 dashed 容器框，左侧统一标层号 + 中英文名 + 一句描述，让读者一眼分得清"我在第几层"。
  4. 节点信息四件套：中文名 / 英文技术名 / 技术栈 / SLA 指标。没有数字的节点不算合格。
  5. 主路径高亮：选一条最典型的请求（如下单），用加粗 + 区别色把它从顶串到底，让读者知道"系统的脊柱"在哪。
  6. 区分边类型：同步 RPC、异步事件、缓存读、DB 写、CDC、监控采集 — 5-6 种边类型足够，用颜色 + 虚实区分，并写在图例里。
  7. 把横切点单独画：K8s / Istio / Prometheus / Jaeger 不属于任何业务层，单独一层贯穿是最干净的画法，别在每个节点里画"还有监控"。
  8. 避免画成"框堆"：节点过多时同领域分组 + 行标签（本图 4 层用"交易/商品/边缘" 3 行），比扁平堆叠更易读。
