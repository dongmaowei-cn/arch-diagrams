微服务架构图的骨架

  1. 切 Tier：client / edge / BFF / core / data 5 层是工业标准 — 横向带 + tier label。
  2. 定核心服务：DDD 限界上下文 → 一个服务一个 box，标 名称 + 技术栈 + QPS + 实例数。
  3. 侧挂 Sidecar：每个服务右上角小 envoy 标记，表示 service mesh 接管流量。
  4. 分线类型：spine（olive 主路径）· sync RPC（橙）· async event（紫虚线）· DB R/W（橙点线）· cache（金虚线）· discover/config（灰/青点线）。
  5. 独立 Platform 列：注册中心 / 配置中心 / Kafka / Saga / 熔断 / 追踪 / 监控 都放右侧独立栏，避免穿插业务服务。
  6. 挂 CORE 徽章：主路径上的关键服务（API Gateway · order-service）挂 olive 实心 CORE 徽章。
  7. 不画太细：物理 pod / 节点 / IP 不画 — 那是部署图。这张是架构图，逻辑视图为主。
  8. 横切 [ex] 角标：熔断 / 限流 / 追踪 / 监控不是 12-factor 标准元素，是工程实践扩展 — 挂金色 [ex] 角标区分。
