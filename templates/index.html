用 9 种标准节点形状(起止 / 处理 / 判断 / I/O / 数据库 / 子流程 / 文档 / 连接符 / 注释)和 4 种线条(默认 / YES / NO / DB)把一个业务流程从触发到终态画出来。每条规则源自 ISO 5807 国际流程图标准。每张流程图都有且只有一个 START、一个或多个 END,每条边走直角(orthogonal routing)。

怎么画 · How to draw

业务流程图的六步骨架

  1. 定义起止:每张流程图都有且仅有一个 START 与至少一个 END。用胶囊形(rounded rectangle)表示触发事件与终态 — 成功用 olive、失败用 rust、中性用 gray 三色区分。
  2. 列出处理动作:把过程中的「动词」抽出来 — 校验、计算、调用、写库、通知。每个动词用一个矩形,标题写动作名,副标写 SLA / 性能指标 / 关键参数。
  3. 标记分支条件:用菱形表示二元判断,出边必须标 YES (olive) / NO (rust) 或具体条件名(如 PASS / FAIL / 等级)。一个菱形只表达一个判断,不要嵌套。
  4. 区分输入输出:用户输入、外部接口回调、消息接收用平行四边形(I/O);持久化存储用圆柱(datastore),读用单向 clay 虚线、写用双向。
  5. 抽出子流程:复杂的子领域(如人工合规审核)用双竖线矩形(subprocess)折叠,避免主图爆炸 — 子流程可以单独画一张图展开。
  6. 处理跨区域跳转:长距离失败路径用连接符(小圆圈 A/B)替代曲线,保持线条永远正交,从而让整图节奏整齐。注释挂方括号引线,不参与流转。
